Keymap

Almost keymaps and printings are defined according to the following rules.

  1. single-line printings: input content with TAP.
  2. double-line printings:
    • bigger top-line contents: input with FLICK.
    • smaller bottom-line content: input with TAP.
  3. top-line printings should keep following rules:
    • 1st character: input to fling to BOTTOM.
    • 2nd character: input to fling to LEFT.
    • 3rd character: input to fling to ABOVE.
    • 4th or later: input to fling to RIGHT, sorted by flick distance in DESCENDING.
  4. UPPER alphabet's flick distance has to be LESS than lower one's flick distance
  5. do NOT show lower in printings for alphabet.
  6. brackets key's printings should keep following rules:
    • need not to keep rule-3.
    • top-line printings should match keymap table.
  7. make its layout closer to physical US layout keyboard AMAP.

Arrow Key

If you tap arrow-key(Arw or Pmv) without flick, you can toggle arrow-key modes.
This key has 2 modes.

  • DEFAULT (Arw): can input direction only with flick(repeatable).
  • PAGE-MOVE (Pmv): can input Page-Up/Down, Home, and End (unrepeatable).

Fn keys and Adjustment

QuickFlick shows keys which can input Fn keys(F1~F12) and can toggle keyboard adjustment in left or right keyboard edge.
You can input following keys(F1-3, F4-6, F7-9, and Fa-c) with tap or fling to left/right, and can change adjustment with up/down flick.
It can also change in settings activity.
(check: Settings > Appearance)
